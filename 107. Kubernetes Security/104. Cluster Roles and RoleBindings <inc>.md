# Cluster Roles and RoleBindings

Objective

The DevOps team is tasked with creating a new Kubernetes user named dev-user and setting up cluster-wide permissions using ClusterRoles and ClusterRoleBindings to provide administrative access across all namespaces.

Task 1: Create a User and Generate a Client Certificate
Create a new user called dev-user and generate the necessary cryptographic materials for authentication. Generate a private key (dev-user.key) and a Certificate Signing Request (CSR) for dev-user. Encode the CSR in base64 and create a Kubernetes CertificateSigningRequest object named dev-user-csr to request signing from the Kubernetes Certificate Authority (CA). This certificate will authenticate dev-user to the Kubernetes API.

Task 2: Create a ClusterRole and ClusterRoleBinding
Define and apply a ClusterRole and ClusterRoleBinding for dev-user that grant administrative permissions. Define a ClusterRole named dev-user-cluster-role with permissions to manage all resources ("*") across all API groups ("*"). Create a ClusterRoleBinding named dev-user-cluster-rolebinding that binds the dev-user-cluster-role to dev-user, thereby granting the necessary permissions to manage the cluster resources.

Task 3: Verify User Permissions
Ensure that dev-user can authenticate with Kubernetes and perform administrative actions as granted by the ClusterRole. Approve the CSR for dev-user and retrieve the signed certificate. Configure kubectl to use dev-user credentials by setting the client certificate and key in the kubeconfig. Set the context to use dev-user and verify the setup by attempting to perform an administrative action, such as listing all resources across all namespaces, to confirm that the permissions are properly applied.
