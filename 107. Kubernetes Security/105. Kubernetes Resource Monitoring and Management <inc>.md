# Kubernetes Resource Monitoring and Management

## Objective

The DevOps team needs to deploy and manage a high-performance web application using Nginx. The team is tasked with ensuring efficient resource utilization and scalability to handle variable workloads.

### Task 1: Deployment Setup and Configuration Review
Successfully deploy Nginx, and review their deployment configurations to ensure it meet the necessary operational requirements. Deploy Nginx with the specified resource requests and limits. The deployment should have 2 replicas, with each pod configured to use a minimum of 100m CPU and 200Mi memory, and limits set to 200m CPU and 400Mi memory.

### Task 2: Monitor Resource Utilization
Monitor and evaluate the resource utilization Nginx deployment to identify any potential optimization opportunities.

### Task 3: Implement Scaling with Horizontal Pod Autoscaler
Implement an automated scaling solution for the Nginx deployment to handle variations in web traffic efficiently. Set up a Horizontal Pod Autoscaler (HPA) for the Nginx deployment. Configure the HPA to adjust the number of pods based on CPU utilization, targeting 50% CPU usage with a minimum of 5 pods and a maximum of 10 pods.

Monitor the HPA status and effectiveness using kubectl get hpa and observe how the number of pods fluctuates in response to changes in load.
