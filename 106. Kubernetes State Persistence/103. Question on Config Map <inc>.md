# Question on Config Map

## Objective
Create a ConfigMap named app-config in the default namespace with a configuration file. Then, create a Pod named config-demo that mounts this ConfigMap as a volume. Use the nginx image and mount the configuration file at /etc/nginx/conf.d.

Nginx Config You Should Use
```
server {
    listen 80;
    server_name localhost;
    location / {
        root /usr/share/nginx/html;
        index index.html index.htm;
    }
    error_page 500 502 503 504 /50x.html;
    location = /50x.html {
        root /usr/share/nginx/html;
    }
}
```

## Tasks

### Task 1: Create the ConfigMap with the Configuration File

Create a ConfigMap: Use kubectl to create a ConfigMap named app-config that includes the nginx configuration file. You can use the --from-file option with kubectl create configmap to include the configuration file.

### Task 2: Create the Pod that Uses the ConfigMap as a Volume

Create a Pod: Define a Pod named config-demo using the nginx image. Configure the Pod to mount the ConfigMap app-config as a volume at /etc/nginx/conf.d.

### Task 3: Verify that the Configuration File is Correctly Mounted

Verify the Mount: Use kubectl exec to access the Pod and check that the nginx configuration file is correctly mounted at /etc/nginx/conf.d. You can use cat or similar commands to view the contents of the configuration file.
